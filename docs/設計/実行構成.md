# DesktopAgent 実行構成

この文書は、[要件.md](C:\Users\kamiy\WorkSpace\desktop-agent\docs\要件.md) を満たすための DesktopAgent の実行構成を定義する。

## 方針

DesktopAgent はローカル完結型アプリケーションとする。

ユーザーの画面情報、会話内容、LLM 入出力、音声合成データ、ログは外部ホスティングサービスへ送信しない。必要な処理はすべてユーザーのローカルマシン内で完結させる。

初期構成では、DesktopAgent 専用の API サーバーは置かない。Unity を制御主体とし、ホスト上の native プロセスまたはローカル Docker Compose 上の依存サービスへ直接接続する。

## 実行形態

DesktopAgent は、Unity 本体、ホスト依存、Docker コンテナ依存を組み合わせて動作する。

- Unity はコンテナ化しない
- Ollama はホスト上の native プロセスとして動作する
- VOICEVOX と MySQL はローカル Docker Compose 上で動作する

実行形態は次の 2 つとする。

- ローカル開発: Unity Editor をホスト上で起動する
- 本番実行: Unity のビルド済み exe をホスト上で起動する

## 構成要素

### Unity

Unity は DesktopAgent のアプリケーション本体であり、Watch と Chat の制御主体である。

Unity は次を担当する。

- 画面取得
- 差分判定
- 文脈管理
- キャラクター表示
- 音声再生
- Ollama、VOICEVOX、MySQL との接続制御

Unity はローカル依存サービスにだけ接続する。

### Ollama

Ollama は LLM 推論を担当する。

- 画面理解
- 作業文脈判定
- Watch リアクション生成
- Chat 応答生成

Unity は `ollama.exe` をホスト上で直接起動し、専用ポートへ接続する。

モデルが不足している場合は、Unity 側で起動後に pull する。

モデル保存先は Ollama の native 保存先を利用する。デフォルトでは exe 配置先ではなく、ユーザーごとの Ollama 管理ディレクトリに保存される。

### VOICEVOX

VOICEVOX は Chat などの明示的な発話テキストの音声合成を担当する。

- Watch リアクションはテキスト表示のみとし、VOICEVOX へ送らない
- 音声再生、発話キュー、割り込み制御は Unity 側で扱う

VOICEVOX は Docker Compose で起動する CPU 版コンテナを利用する。

### MySQL

MySQL は設定、ログ、将来の永続データの保存先として用意する。

## ホスト依存

DesktopAgent を現在の構成のまま動かす場合、ホスト側には少なくとも次が必要である。

- Windows 上で Unity Editor または Unity ビルド済み exe を実行できること
- `ollama.exe` が利用可能であること
- Docker Desktop と `docker compose` が利用可能であること

ホストへ直接インストールが必要なのは Ollama と Docker 実行環境である。

MySQL と VOICEVOX はアプリケーションの依存要素ではあるが、ホストへ直接インストールする前提ではなく、Docker Compose によってコンテナとして起動する。

## dev/prd 構成

開発用 `dev` と運用用 `prd` は接続先ポートと Docker volume を分離する。

ポート割り当ては、native プロセスを `10001` から `10010`、Docker コンテナを `10011` 以降に寄せる。prd では同じ規則を `20001` からの帯で使う。

### dev

dev は Unity Editor から利用する。

- Compose プロジェクト名: `desktop-agent-dev`
- Ollama 接続先ポート: `10001`
- VOICEVOX 公開ポート: `10011`
- MySQL 公開ポート: `10012`
- MySQL volume: `desktop-agent-dev-mysql-data`

### prd

prd は Unity のビルド済み exe から利用する。

- Compose プロジェクト名: `desktop-agent-prd`
- Ollama 接続先ポート: `20001`
- VOICEVOX 公開ポート: `20011`
- MySQL 公開ポート: `20012`
- MySQL volume: `desktop-agent-prd-mysql-data`

### 分離ルール

dev と prd は次を分離する。

- Compose プロジェクト名
- 公開ポート
- MySQL 用 Docker volume
- MySQL データ
- ログ
- Unity 側の接続設定

Ollama のモデルデータは native Ollama の保存先を使うため、Docker の dev/prd 構成では分離しない。

## セキュリティとプライバシー

DesktopAgent はローカル完結を前提とする。

- Unity はローカル依存サービスにだけ接続する
- Ollama はホスト上の native プロセスとして動作する
- VOICEVOX と MySQL はローカル Docker Compose 上で動作する
- 画面キャプチャ画像は外部へ送信しない
- LLM 入出力は外部へ送信しない
- 音声合成テキストと音声データは外部へ送信しない
- ログは外部へ送信しない

公開ポートは Unity からのローカル接続のために利用し、外部ネットワークへの公開を目的としない。

## 将来の見直し

初期構成では API サーバーを置かない。

ただし、次のような条件が出た場合は実行構成を再検討する。

- Unity から直接扱うには重い処理が増えた場合
- 永続化やログ分析を専用プロセスへ分離したい場合
- Docker 依存を減らして配布しやすくしたい場合

その場合でも、原則としてユーザーのローカルマシン内で完結する構成を維持する。
