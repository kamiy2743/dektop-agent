# DesktopAgent 要件概要

## 目的

DesktopAgent は、ユーザーのPC作業を画面から観察し、作業状況に応じて軽い反応を返すデスクトップ常駐型のキャラクターアプリケーションです。

## 用語

- Watch: 観察処理。画面キャプチャ、画像認識、ユーザー作業の軽い反応を扱う常駐処理。
- Chat: チャット処理。ユーザーが明示的に話しかけたときの重い会話処理。

## 機能要件

### 観察処理

- アプリケーションは、ユーザーが選択したモニターを定期的に観察できること。
- アプリケーションは、画面内容に大きな変化がない場合、不要な画像認識を抑制できること。
- アプリケーションは、画面が静止している場合でも、一定時間ごとに再観察できること。
- アプリケーションは、古い画面に対する認識結果が後から返ってきた場合、現在の画面状態と矛盾しないよう扱えること。

### 画面理解

- アプリケーションは、現在画面の中心的な作業内容を説明できること。
- アプリケーションは、画面上の中心対象を汎用名ではなく、できるだけ区別可能な名前や特徴として扱えること。
- アプリケーションは、画面に見えていない情報を推測せず、観察可能な内容に基づいて判断すること。
- アプリケーションは、作業カテゴリ、注目対象、注目内容、読めるテキストなどを観察結果として扱えること。

### 作業コンテキスト判定

- アプリケーションは、現在の観察結果と直近の作業文脈を比較し、作業の継続、詳細化、転換、割り込みなどを判定できること。
- アプリケーションは、反応すべき変化かどうかを、重要度や新規性に基づいて判断できること。
- アプリケーションは、短時間のノイズや判定不能な観察に対して、不必要な反応を抑制できること。
- アプリケーションは、直近の観察履歴を保持し、作業文脈を継続的に更新できること。

### リアクション

- アプリケーションは、Watch 判定に基づいて短いテキストリアクションを生成できること。
- Watch リアクションは、システム通知や分析メモではなく、キャラクターがそばで見ているような自然な一言であること。
- Watch リアクションは、最新の観察内容に基づき、過去画面にしかない情報を混ぜないこと。
- Watch リアクションは、同じ内容や同じ言い回しを繰り返しにくいこと。

### チャット処理

- アプリケーションは、Watch とは別に、ユーザーが明示的に話しかけたときの Chat 応答を扱える構成であること。
- Chat 応答は、Watch より重い会話処理として扱い、必要に応じて現在の作業文脈を参照できること。

### 音声

- アプリケーションは、キャラクターの発話を音声合成できる構成であること。
- アプリケーションは、発話中に新しい発話が発生した場合でも、再生順や割り込み方を制御できること。
- ユーザーは音声をミュートできること。

### UI

- ユーザーは、現在の状態、認識対象の画面、画面説明、Watch 判定、Watch リアクションを確認できること。
- ユーザーは、対象モニター、観察間隔、判定しきい値、利用モデル、キャラクタープロンプトなどを調整できること。
- ユーザーは、キャラクター表示の有無を切り替えられること。

### ログ

- アプリケーションは、観察処理、認識、判定、リアクション生成、エラーをログとして記録できること。
- ログは後から調査できる場所に保存されること。
- ログ確認を行う場合は、元ログを直接読むのではなく、専用の分析手順を通して確認すること。

## 外部連携要件

- 画像認識、作業判定、リアクション生成、Chat 応答にはローカルLLMを利用できること。
- ローカルLLMの接続先や使用モデルは設定可能であること。
- 音声合成にはローカル音声合成エンジンを利用できること。
- 外部連携のタイムアウトやモデル保持時間は調整可能であること。

## 非機能要件

- 観察処理は常駐してもユーザー作業を過度に妨げないこと。
- 画面変化が少ない場合は、不要な推論を抑えて負荷を下げること。
- 反応は短く、頻繁すぎず、作業の邪魔にならないこと。
- 認識結果やリアクションは、現在画面との整合性を優先すること。
- ユーザーの画面内容を扱うため、見えていない個人情報や意図を推測しないこと。
